## 🤖 Automating This Profile

<details>
<summary><b>Click to see how the dynamic sections stay up to date automatically</b></summary>
<br>

This profile uses **three layers of automation** so it never goes stale:

**1. Live widgets (zero setup)**
The Stats, Top Languages, Streak, Trophy, Activity Graph, Quote, Joke, and Profile View Counter images above are all rendered live on every page load by free hosted services (`github-readme-stats`, `streak-stats`, `github-profile-trophy`, `github-readme-activity-graph`, `komarev`). Nothing to schedule — they always reflect your current GitHub data.

**2. Contribution Snake (GitHub Action)**
Create `.github/workflows/snake.yml` in a repo named exactly like your username (`theghq/theghq`) with this content:

```yaml
name: Generate Snake Animation

on:
  schedule:
    - cron: "0 */6 * * *"
  workflow_dispatch:
  push:
    branches: [ main ]

jobs:
  generate:
    permissions:
      contents: write
    runs-on: ubuntu-latest
    steps:
      - uses: Platane/snk@v3
        with:
          github_user_name: theghq
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark
      - uses: crazy-max/ghaction-github-pages@v4
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**3. Latest blog posts (Dev.to / Medium / personal RSS)**
Add a markdown placeholder block to this README, then let an Action fill it in:

```markdown
<!-- BLOG-POST-LIST:START -->
<!-- BLOG-POST-LIST:END -->
```

Then create `.github/workflows/blog-post-workflow.yml`:

```yaml
name: Latest Blog Post Workflow

on:
  schedule:
    - cron: "0 */12 * * *"
  workflow_dispatch:

jobs:
  update-readme:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: gautamkrishnar/blog-post-workflow@v1
        with:
          feed_list: |
            https://dev.to/feed/theghq
            https://medium.com/feed/@theghq
          max_post_count: 5
```

Swap the feed URLs for your real Dev.to / Medium / blog RSS feeds. This Action runs on a schedule, fetches your latest posts, and rewrites the placeholder block automatically — no manual edits ever needed.

**Recent commits / PRs / issues / stars**
These can be surfaced the same way using `lowlighter/metrics`, a single Action that generates an always-fresh SVG covering recent activity, PRs, issues, and starred repos in one image — drop its output between a comment-marker block exactly like the blog workflow above.
